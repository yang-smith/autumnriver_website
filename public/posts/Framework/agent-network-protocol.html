Agent网络协议 v0.1

二〇二六年四月

基本概念

节点：参与网络的实体Agent。能力写在description里。

中继节点：承担广播职责的节点Agent。接受WebSocket连接，将所有收到的消息转发给所有其他连接节点，分配账本序号。中继节点是网络唯一的中心化基础设施，其自身也是网络中的一个普通节点。

账本：所有节点的能力描述集合。每个节点持有完整副本。

账本格式

<address> | <description> [@ <ws-address>]
  • address：公钥的hash。公私钥推荐使用 Ed25519
  • description：Agent的自然语言描述
  • ws-address（可选）：WebSocket连接地址，仅对可连接节点（如中继节点）有意义

消息格式

message（节点→节点，经中继广播）

{
  "from": "<address>",
  "to": "<address> | *",
  "content": "<自然语言>"
}

第一逻辑：中继节点负责将message广播给所有连接者

额外逻辑：当 to = '*'，这就是一条注册/更新消息，中继节点接受后，发现新的，进行注册然后额外发给他；中继节点拥有的账本，如果已经存在则自己更新账本。

约定：context带有

<address>..</address>
<description>..</description>
<address>..</address>
<description>..</description>...

结构的就是账本更新。

节点规范

加入网络

  1. 通过带外方式获取中继节点地址，建立WebSocket连接
  2. 广播 message（to: *）注册自己并获取账本
  3. 收集在线节点回复，更新本地账本

发送消息

  • 从本地账本选取目标节点
  • 发送 message

接收消息

  • message：校验 to 是否为 * 或等于自己的address。匹配则处理，否则丢弃。如何处理，节点自主决定。
